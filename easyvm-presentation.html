EasyVM

Automatisierte VM-Erstellung im Azure DevTest Lab

Assecor GmbH · Juni 2026

Das Problem

Aktuell muss jeder Benutzer, der eine neue virtuelle Maschine benötigt, mehrere manuelle Schritte im Azure Portal durchlaufen.

Heute (Ist-Zustand)

1. Benutzer öffnet unsere App

2. Weiterleitung zum Azure Portal

3. VM manuell erstellen

4. Disks hinzufügen

5. Netzwerk konfigurieren

6. Artifacts installieren

7. Weitere Einstellungen ...

Nachteile: Fehleranfällig, zeitintensiv, inkonsistente Konfigurationen, erfordert Azure-Kenntnisse
Ziel (Soll-Zustand)

1. Benutzer öffnet unsere App

2. Einfaches Formular ausfüllen

(VM-Zweck, Größe, OS ...)

3. Klick auf „Erstellen“

4. Alles wird automatisch provisioniert

Vorteile: Schnell, einheitlich, keine Azure-Kenntnisse nötig, reproduzierbare Umgebungen

Entscheidungsmatrix

Kernfrage: Nutzen oder planen wir die Nutzung eines nicht-Azure Cloud-Providers (AWS, GCP)?
Diese Antwort bestimmt maßgeblich die Tool-Wahl.
Wenn ...Dann ...
Assecor ist und bleibt Azure-only, Backend ist .NET Bicep Geringster Aufwand, native Integration, kein Overhead
Terraform wird bereits in anderen Projekten eingesetzt Terraform Konsistenz über Teams wichtiger als Einrichtungsaufwand
Multi-Cloud (AWS/GCP) ist ein realer, geplanter Bedarf Terraform Investition in Portabilität zahlt sich langfristig aus
Komplexe Orchestrierung nötig (Genehmigungsflows, bedingte Provisionierung) Bicep + SDK Bicep für Templates, SDK für Steuerungslogik

Geschätzte Arbeitseinsparung

Bicep spart ca. 40–60% Implementierungsaufwand vs. Terraform

Bicep spart ca. 70–80% Implementierungsaufwand vs. reinem SDK

Fazit

Der Kostenunterschied liegt nicht in der Lizenzierung — alle Tools sind grundsätzlich kostenlos. Der entscheidende Faktor sind die Entwicklerstunden für Implementierung und Wartung.

Umsetzungsphasen

1

Templates definieren

Parametrisierte Bicep-Templates für jeden VM-Zweck erstellen (Name, Größe, OS, Disks, Subnet, Region, Tags)

2

Backend-Integration

Azure SDK für .NET nutzen, Formular-Input auf Template-Parameter mappen, Deployment auslösen

3

Netzwerk & Sicherheit

VNet/Subnet-Zuordnung, NSG-Regeln, RBAC-Rollen, Azure Policy für Guardrails

4

Post-Provisioning

VM-Extensions, Artifacts, Software-Installation, Statusbenachrichtigung an Benutzer
